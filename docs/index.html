TSLab Cloud Server

TSLab на вашем VPS — одной командой

Поставьте торговый сервер TSLab на собственный Linux-сервер и подключите его к своему аккаунту TSVerse. Без навыков системного администратора.

🐧 Linux / 🍏 macOS / 🪟 Windows 🐳 Docker 🔒 Вход через TSVerse (QR) ♻️ Данные сохраняются

Что нужно

VPS или сервер

Любой Linux-сервер (Ubuntu/Debian и др.), 1–2 ГБ RAM. Подойдёт и домашний компьютер с Docker.

Доступ по SSH

Обычный пользователь с правами sudo (или root). Так заходят на VPS у любого хостинга.

Аккаунт TSVerse

Ваш аккаунт на tsverse.pro. Подтверждение входа — с телефона по QR.

Docker

Linux: скрипт ставит Docker сам. Windows: нужен Docker Desktop — скрипт попробует поставить его через winget, но после установки потребуется перезагрузка и запуск Docker Desktop.

1 Установка одной командой

Зайдите на сервер по SSH и вставьте команду. Она проверит/поставит Docker, скачает TSLab, запустит его и доведёт до входа в TSVerse.

Linux / macOS
Windows (PowerShell)
curl -fsSL https://nektodron.github.io/tslab-cloud/install.sh | bash
irm https://nektodron.github.io/tslab-cloud/install.ps1 | iex
⚠️ Windows / PowerShell: используйте вкладку «Windows (PowerShell)» выше (команду irm … | iex). Команда curl … | bash в PowerShell не работает — там curl это Invoke-WebRequest, и вы получите ошибку «Не удается найти параметр fsSL».
Контейнер слушает только localhost — наружу ничего не открывается. Управлять сервером вы будете из своего обычного TSLab на компьютере: в нём выбираете удалённый сервер, и команды идут на VPS через TSVerse. Открывать веб-интерфейс сервера в интернет не нужно.

2 Вход в TSVerse (по коду / QR)

Через ~30–60 секунд после старта (сервер сперва пытается восстановить прошлую сессию) скрипт покажет короткий код, ссылку и QR-код прямо в терминале:

────────────────────────────────────────────────
  Вход в TSVerse
  1) Откройте на телефоне:  https://id.tsverse.pro/device
  2) Введите код:           WDJB-MJHT
  (или отсканируйте QR-код ниже)
────────────────────────────────────────────────
  █▀▀▀▀▀█ ▀▄█▀▄ █▀▀▀▀▀█
  █ ███ █ █▀ ▄▀ █ ███ █
  █ ▀▀▀ █ ▄█▀█▄ █ ▀▀▀ █
  ▀▀▀▀▀▀▀ █▄▀▄█ ▀▀▀▀▀▀▀
  ...
  Waiting for confirmation...
  1. Отсканируйте QR-код телефоном (или откройте ссылку и введите код вручную).
  2. Войдите в свой аккаунт TSVerse и нажмите «Подтвердить».
  3. Скрипт сам поймёт, что вход выполнен, и напишет «Подключено к TSVerse».
Не видите код? Он появляется не сразу — дождитесь блока с QR в логах (используйте -f, чтобы они не обрывались):
docker logs -f tslab
Не успели за 5 минут — код просто обновится, новый будет там же.
После первого входа сервер запоминает вас. При перезапуске и обновлении повторно входить не нужно — сессия восстанавливается автоматически.

3 Управление через браузер

Сервером удобнее всего управлять из веб-панели — её не нужно устанавливать, она открывается в любом браузере с любого устройства и общается с вашим VPS только через TSVerse (порты наружу открывать не надо).

  1. Откройте панель управления:
    🖥️ Открыть панель управления →
    Прямая ссылка: nektodron.github.io/tslab-cloud/app/
  2. Нажмите «Войти через Identity Server» и авторизуйтесь под тем же аккаунтом TSVerse, в который вошёл ваш сервер на шаге 2.
  3. Ваш сервер выберется автоматически. Если серверов несколько — выберите нужный в списке «Target» вверху страницы.
  4. Готово — управляйте агентами, скриптами, графиками и сделками. Все команды идут на ваш VPS через TSVerse.
Панель ничего не хранит у себя: вход — это ваш токен TSVerse в браузере, а данные и команды идут напрямую на ваш сервер.

Служебное управление

Кроме веб-панели (шаг 3), сервером можно управлять и из вашего обычного TSLab на компьютере — выберите удалённый сервер, команды пойдут на VPS через TSVerse. Ниже — команды на самом сервере по SSH (логи, перезапуск, обновление):

Команды на сервере (по SSH)

docker logs -f tslab      # логи (и код входа в TSVerse)
docker restart tslab      # перезапуск (вход сохранится)
docker stop tslab         # остановить
docker rm -f tslab        # удалить контейнер (данные останутся)

Обновление до новой версии

Просто запустите установку ещё раз — подтянется свежий образ, данные и вход в TSVerse сохранятся:

curl -fsSL https://nektodron.github.io/tslab-cloud/install.sh | bash

Данные и безопасность

  • Где данные. Всё (база, скрипты, настройки, сессия TSVerse) хранится на сервере в папке ~/.local/share/tslab, а не внутри контейнера — поэтому переживает перезапуск и обновление.
  • Бэкап. tar czf tslab-backup.tgz -C ~/.local/share/tslab .
  • Управление — через ваш локальный TSLab. В обычном TSLab на вашем компьютере выберите удалённый сервер — запросы пойдут на VPS через TSVerse. Веб-интерфейс самого сервера для этого не нужен: он слушает только 127.0.0.1:8088 и наружу не открыт.
  • Прямой доступ к WebUI сервера (необязательно). Если всё же хотите открыть его напрямую — пробросьте по SSH: ssh -L 8088:localhost:8088 user@your-vps, затем http://localhost:8088.
  • Вход. Подтверждение входа всегда происходит на вашей стороне (телефон/браузер) — пароль в скрипт и на сервер не передаётся.

Если что-то пошло не так

Docker не ставится / нет прав
Нужны права sudo или root. На большинстве VPS вы заходите под root либо под пользователем с sudo. Установить Docker вручную: docs.docker.com/engine/install, затем запустите команду установки снова.
Порт 8088 уже занят
Запустите с другим портом:
TSLAB_PORT=8090 bash -c "$(curl -fsSL https://nektodron.github.io/tslab-cloud/install.sh)"
Windows: ошибка «Не удается найти параметр fsSL» (PowerShell)
Вы вставили Linux-команду в PowerShell. Возьмите команду из вкладки Windows (PowerShell):
irm https://nektodron.github.io/tslab-cloud/install.ps1 | iex
Не увидел код входа в TSVerse
Код появляется через ~30–60 секунд после старта (сервер сперва пытается восстановить прошлую сессию). Смотрите логи с -f и дождитесь блока с QR: docker logs -f tslab. Если вход уже был выполнен ранее, кода не будет — сессия восстановлена автоматически.
Как полностью переустановить «с нуля»
Удалите контейнер и данные, затем установите заново:
docker rm -f tslab && rm -rf ~/.local/share/tslab
Файлы в папке данных принадлежат root
Контейнер работает от root, поэтому файлы в ~/.local/share/tslab создаются от root. На своём VPS это нормально. Управлять/удалять их можно через sudo.
